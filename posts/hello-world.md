This is my first post. The blog is live.

## Why a terminal portfolio?

Most portfolios feel like brochures — static, polished, and forgettable. I wanted something that felt like a *tool*. Something you interact with rather than scroll through.

## How it works

Posts are plain markdown files in the `/posts` folder of this repo. The site fetches them directly from GitHub at runtime — no rebuild, no redeploy. I push a file and it's live.

```
write post.md → git push → live
```
